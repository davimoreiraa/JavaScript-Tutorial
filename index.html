Variáveis

As variaveis sao nomes simbolicos para guardarmos valores, podemos declarar variaveis de 3 diferentes maneiras:
  1. •var
  2. •let
  3. •const --> Não permite que o valor atribuido a ela seja mudado.
Exemplos: 

            var clima = 'quente'
            console.log(clima) // 'quente'
Podemos descobrir o tipo de uma variavel usando o typeof.
Ex: 

            console.log(typeof clima) //imprimirá string
O JS é uma linguagem fracamente typada, isso significa que ela é bem dinâmica e não muito exigente em sua escrita.

Escopo

Escopo --> visibilidade de uma variavel, cada variavel possui um escopo diferente
podemos criar um Bloco de escopo envolvido em {}, e dentro dele, os valores serão locais, e cada variavel lida de maneira diferente com a dinamica desse bloco
escopo var --> ao declarar uma variavel usando o var, observamos uma caracteristica do js, o hoisting. O JS lê o código verticalmente, porem, a declaração de variaveis é elevada, como se fossem antes de tudo.
OBS: apenas a sua declaraçõa sofre esse efeito, a atribuição dela não.
Ex: 


              console.log('O valor de x é', x)
              var x = 2
Nesse caso, o terminal imprimirá 'o valor de x é undefinied' , pois ele elevou a declaração da variavel x, mas nao a sua atribuição de valor.
escopo let --> diferentemente do var, o let não sofre o efeito de hoisting, e alem disso, so funciona no escopo onde foi criada. 

               let y = 1

                {
                 let y = 2
                 console.log(y) //imprimirá 2
                }

                console.log(y) //imprimirá 1
Depedendo de onde o y, foi criado, ele pode ter diferentes valores. Porém, podemos dentro de um bloco alterar o valor de uma variavel let declarada fora. 

              let y = 1

              {
                 y = 2
              }

              console.log(y) //imprimirá '2'

Function

As funções sofrem o efeito de hoisting
As funções tambem possuem escopo
Exemplo: 


          let fruta = 'maça'

          function comprarFruta(fruta) {
            fruta = 'laranja'
            return fruta
          }

          console.log(fruta) // imprimirá maça
          console.log(comprarFruta(fruta)) //imprimira laranja
Apenas dentro da função a variavel sofreu alteração. Porém, quando não há paramêtros na função e alteramos o valor de uma variavel, ela é mudada tambem fora da função.
Exemplo: 

        let fruta

        function comprarFruta() {
           fruta = 'laranja'
           return fruta
        }

        console.log(fruta) // imprimirá undefinied, pois apenas declaramos a variavel, nao atribuimos valor
        console.log(comprarFruta()) //imprimirá laranja

Type conversion x Type coersion

O type conversion é quando alguém altera o tipo de um dado manualemente. Enquanto que o Type coersion é quando essa alteração é feita pelo computador.

Exemplos: 

              console.log(Number('9') + 5) 
              //O número 9 é uma string, porém alteramos manualmente seu tipo para number, o console imprimirá 95

             console.log('9' + 5) // O console imprimirá 95 , pois o 5 foi forçadamente alterado para uma string

Comparação

== -> para comparar duas variaveis, devolve true se forem iguais
!= -> devolve true se forem diferentes
=== -> devolve true se forem iguais em tipo e em valor
!== -> devolve true se tiverem um tipo diferente
Exemplos: 


              let one = 1
              let two = 2

              console.log(one == 1) //true
              console.log(one != 2) //true
              console.log(two != 2) //false
              console.log(one === '1') //false
              console.log(one === 1) //true
              console.log(two !== '2') //true
              console.log(two !== 2) //false

Throw e try catch

Essa é uma maneira de rodarmos uma função.
Exemplos: 

            function getRectArea(width, height) {
              if (isNaN(width) || isNaN(height)) {
                throw 'Parameter is not a number!'
              }
            }
            
            try {
              getRectArea(3, 'A')
            } catch (e) {
              console.error(e)
              // expected output: "Parameter is not a number!"
            }

Classes

No JS, as classes são como moldes para criarmos objetos. Nela, podemos ter métodos, que são funções como propriedades de objetos. É muito comum que nelas tenhamos um método chamado constructor() , ele serve para declarar e adicionar valores às propriedades. Exemplo: 


            //criando a classe

            class identidade {
            constructor(nome,cpf,anoNascimento) this.nome = nome this.cpf = cpf
            this.anoNascimento = anoNascimento } 


            //criando nova variável moldada na classe feita 

           var identidadeDavi = new identidade("Davi",999,2005)

É possível também criarmos a extensão de uma classe. Essa nova extensão terá todas as propriedades da classe-mãe, e ainda podemos adicionar mais algumas. 


            class cartaoSaude extends identidade {
              constructor(endereco, vacina)
            }

Em uma extensão, existe o super() usado para chamarmos uma propriedade da classe mãe. Exemplo: 


            class cartaoSaude extends identidade {
              constructor(endereco, vacina)
              super(nome,cpf)
            }

Caso não chamamos a propriedade da classe-mãe usando o super(), podemos sobrescrever uma propriedade. 


            class cartaoSaude extends identidade {
              constructor(endereco, vacina)
              super(nome,cpf)
            }
